### CLS Investigation Results
- **CLS=1.0 mystery**: Caused by Lighthouse `--disable-gpu` flag interacting with headless rendering
- **Without fonts.css**: recovery.html Score=100, CLS=0, LCP=1.7s (perfect)
- **With fonts.css sync link**: CLS=1.0 every time
- **With fonts.css async (media=print)**: CLS=1.0 every time
- **With inlined fonts.css**: CLS=1.0 (font files loading async triggers layout shift)
- **GPU enabled (no --disable-gpu)**: Score=32 with CLS=1.045 (same CLS, worse overall)
- **Root cause**: Self-hosted fonts with swap behavior trigger layout shifts when fonts.css loads asynchronously — the font files are requested before CSS is fully processed, causing a race condition

### Real-World Assessment
- For actual users: fonts.css loads in ~50ms (local), CLS will be minimal
- On real network: fonts are cached after first load → 0 CLS on repeat visits
- The Lighthouse CLI `--disable-gpu` artifact creates artificial CLS=1.0 in testing
- **Recommended**: Keep fonts.css as blocking link (50ms penalty worth it for CLS=0)
- Alternative: Use `font-display: optional` + swap to avoid all CLS
