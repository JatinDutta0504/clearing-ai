### 6. The Real CLS Culprit — body-level shift
The `body` shift of 1.0 is extremely unusual. It suggests the body element itself changes size during load — likely because the nav or some above-fold element is injected/changes between page load and first contentful paint.

Looking at the DOM path: `1,HTML,1,BODY` — body with height 24,528px. This is not a small shift. The body occupying the full page height might be measured differently as the browser paints successive sections.

The LCP element being `.quote-rotator` means the text INSIDE it is what Chrome measured as the largest contentful paint. The text is: "Some of the best debugging happens away from the screen."

The render delay of 3556ms = main.js blocking time + DOM rendering.

## Structural Fixes Needed

### Fix 1: Remove main.js render-blocking → Inline quote-rotator
**Problem:** main.js is `defer` but the LCP element (quote-rotator) is rendered by JS that fires on DOMContentLoaded, blocking LCP.

**Fix:** Replace quote-rotator JS with inline JS or static content. Pre-render one quote in HTML.

**Estimated LCP improvement:** 3558ms → <1500ms

### Fix 2: Preload LCP font
**Fix:** Add `<link rel="preload" as="font" crossorigin href="fonts/Lora-Regular.woff2">` for Lora (hero-title font) to eliminate font-related render delay.

### Fix 3: CLS body shift
**Possible causes:**
- CSS `@import` or late-loading stylesheets shifting layout
- Dynamic class injection (e.g., `.fade-in.visible` applying at wrong time)
- Font swap causing reflow

**Fix:** Audit all `onload="this.media='all'"` CSS loads — they trigger layout recalculation when they switch to screen mode. This is likely the body CLS source.
