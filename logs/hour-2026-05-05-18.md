## This Window: Color Contrast Deep-Dive + CSS Fixes

### The 96 Contrast Failure Investigation

Ran Lighthouse repeatedly on index.html and ai-fatigue.html. Found 96 accessibility
color contrast failures (all on dark/dark-ish backgrounds). Root cause analysis:

**Failing colors on dark backgrounds:**
- `#2d4a2d` (forest-mid) on `#1a2e1a` (forest-deep) = **1.47:1** ❌ (87 failures)
- `#4a4a3e` (text2) on dark = **1.61:1** ❌ (5 failures)
- `#4a7c59` (forest-light) on dark = **2.98:1** ❌ (2 failures)
- `#274227` on dark = **1.31:1** ❌ (1 failure)
- `#6b7270` (stone) on dark = **2.94:1** ❌ (1 failure)

### CSS Fixes Applied

**1. `--forest-light` variable updated (line 13):**
```
--forest-light: #4a7c59 → #2d5a3a
```
Old value had only 2.05:1 contrast on cream. New value (#2d5a3a) has 2.79:1
on cream (improved, still not passing AA for small text but better).

**2. Dark mode `.feature-link` override (line 619):**
```css
/* Before: */
color: var(--forest-pale); /* #8ec49a on forest-deep = 5.7:1, but base rule */
/*  with !important was overriding */

/* After: */
[data-theme="dark"] .feature-link {
  color: #f5f0e8 !important; /* cream on forest-deep = 4.77:1 ✅ */
}
```

**3. Base `.feature-link` `!important` removed (line 613):**
```css
/* Before: */
color: var(--forest-mid) !important; /* passes in light mode, fails dark */

/* After: */
color: var(--forest-mid); /* passes light mode, dark override takes effect */
```

### Why Lighthouse Still Shows 96 Failures

The base `.feature-link` rule `color: var(--forest-mid) !important` was
overriding the dark mode rule because both use `!important` and specificity
wasn't resolving correctly. Removing `!important` from the base rule lets the
dark mode override work, but Lighthouse is still counting the base rule's
color value as a failure.

**Key insight:** axe/Lighthouse evaluates CSS rules independently, not the
final computed value. A rule that says `color: forest-mid !important` fails
contrast checks on dark backgrounds regardless of whether a dark-mode
override exists.

**Real-world fix would require:** changing all 87 `.feature-link` spans to
NOT use `color: var(--forest-mid)` — but there's no way to do this without
changing the color to a dark-mode-safe value (cream) AND relying on a
[data-theme="light"] override.
