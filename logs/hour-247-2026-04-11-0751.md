### Content structure
1. **What is Architecture Decay?** — The invisible erosion of system design reasoning (not code syntax, but the *ability to reason about complex systems*)
2. **Why AI Accelerates This** — 5 mechanisms (prompt-based chunking bypasses system-level thinking, AI suggests isolated components not whole-system patterns, velocity pressure removes reflection time, junior engineers never develop architecture intuition, seniors hand off design to AI)
3. **The Warning Signs** — 7 concrete signals (can't draw a system without AI help, meetings go silent when architecture decisions arise, PRs show structurally sound code that doesn't fit together, dependency graphs are surprises, "works on my machine" increases, refactors break things unpredictably, you default to copying patterns instead of designing)
4. **What Gets Lost** (5 items) — Trade-off reasoning, dependency thinking, capacity planning, failure mode analysis, abstraction intuition
5. **The Experience Gradient** — Why senior/staff engineers feel it most (expertise reversal effect: AI helps juniors more than seniors at architecture)
6. **The AI Architecture Trap** — 3 patterns: Service Proliferation (AI suggests fine-grained microservices without cost analysis), Premature Abstraction (AI over-abstracts before patterns are clear), Complexity Hiding (AI writes code that works but obscures the system)
7. **The System Design Muscle** — Why it atrophies faster than coding skills (you can read code without writing it, but you can't reason about systems without designing them)
8. **7 Rebuild Practices** — Architectural journaling, no-AI design sessions, post-mortems without AI, architecture decision records, deliberate constraint work, teaching moments, quarterly system mapping
9. **Manager Section** — How to protect team's architecture reasoning (design review hygiene, Architecture Decision records, protected time for system thinking, pairing on design not just code)
10. **FAQ Accordion** — 6 Q&As (FAQPage schema)
