### Bug: performance-review-ai-fatigue.html Missing Google Fonts

**Problem:** Page uses `css/style.min.css` which defines `--font-serif: 'Lora'` and `--font-sans: 'Inter'`.
But the HTML head had no `<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=...">` tag.

**Impact:**
- Lora + Inter would NOT load from Google Fonts
- Browser uses system font fallbacks (Georgia/system-ui) instead of intended Lora/Inter
- CLS risk: system fonts may have different metrics than Lora/Inter → layout shift
- Visual inconsistency: page looks different from rest of site

**Fix applied:**
```html
<!-- Added after preconnects: -->
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Lora:...&family=Inter:...&display=swap"
      media="print" onload="this.media='all'" />
<noscript><link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Lora:...&family=Inter:...&display=swap" /></noscript>
```

Pattern: Non-render-blocking (media="print" onload trick) — same as best-performing pages on site.
