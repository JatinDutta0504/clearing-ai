## What was done

Ran Lighthouse audits on 6 key pages (index, manifesto, recovery, explanation-requirement, quick-start, r-programming-llm-ban). Found CLS = 1.003 on index (target: <0.1), primarily caused by Google Fonts.

### Root cause analysis:
1. **Render-blocking Google Fonts** — 143 pages had `<link rel="stylesheet">` pointing to fonts.googleapis.com — this blocks rendering and causes layout shifts when fonts load
2. **display=swap** — Using `&display=swap` means font-swap causes visible CLS when custom fonts replace system fonts

### Fixes applied (2-pass Python script):

**Pass 1:** Convert all render-blocking Google Fonts links to async loading:
- Pattern: `<link rel="stylesheet" href="...googleapis.com/css2...">`
- Changed to: `<link rel="stylesheet" href="...googleapis.com/css2..." media="print" onload="this.media='all'" />`
- Fixed 145 render-blocking links across 123 pages
- Plus 67 remaining links in second pass

**Pass 2:** Replace `&display=swap` with `&display=optional` in all 315 Google Fonts hrefs:
- `display=optional` uses metric-compatible fallbacks — no visible swap, zero CLS from font loading
- Fixed 315 links across 139 pages
