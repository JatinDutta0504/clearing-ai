## Phase 4 Community Window 97 — Newsletter Infrastructure Fix (Formspree mailto Fallback)

**Context:** 13+ files have `YOUR_FORM_ID` placeholder blocking newsletter capture. Formspree requires human login/account creation — not automatable. **Solution: Upgrade all newsletter forms to auto-detect YOUR_FORM_ID and use mailto fallback when Formspree isn't configured.**

### What was fixed:

**Pattern applied across 13 files:**
```javascript
// Newsletter form — Formspree when configured, mailto fallback otherwise
const form = document.getElementById('newsletterForm');
if (form) {
  form.addEventListener('submit', e => {
    if (form.action.includes('YOUR_FORM_ID')) {
      e.preventDefault();
      const email = form.querySelector('[name="email"]').value;
      const name = form.querySelector('[name="name"]') ? form.querySelector('[name="name"]').value : '';
      const subject = encodeURIComponent('Subscribe to The Dispatch 🌿');
      const body = encodeURIComponent((name ? 'Name: ' + name + '\n' : '') + 'Email: ' + email + '\n\nPlease subscribe me to The Dispatch — the weekly letter for AI-fatigued engineers.');
      window.location.href = 'mailto:hello@clearing-ai.com?subject=' + subject + '&body=' + body;
      const btn = form.querySelector('button');
      const original = btn.textContent;
      btn.textContent = '✉️ Check your email app';
      btn.disabled = true;
      setTimeout(() => { btn.textContent = original; btn.disabled = false; }, 4000);
    }
  });
}
```
