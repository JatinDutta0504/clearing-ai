## Critical Bugs Found & Fixed

### Bug 1: ai-brownout.html — LCP 2864ms (FAIL)
**Root cause:** Google Fonts `<link>` was placed AFTER inline `<style>` block (~line 103), meaning browser discovered fonts too late. Combined with 2602ms element render delay (font swap FOUT).

**Fix:** Moved Google Fonts to `<head>`, added `<link rel="preload">` for early discovery, removed duplicate post-inline font link.

### Bug 2: code-review-culture.html — CLS 0.402 (FAIL)
**Root cause:** `<link rel="stylesheet" media="print" onload="this.media='all'">` causes browser to initially render with NO CSS, then repaints when CSS loads — causing entire-body layout shift.

**Fix:** Replaced with `<link rel="preload" as="style" onload="this.rel='stylesheet'">` — browser discovers and fetches CSS early without blocking render.

### Bug 3: 8 pages using `href="style.css"` (non-existent file)
**Pages affected:** ai-code-review-fatigue.html, coders-block.html, first-year-engineer-ai-fatigue.html, hn-visitor-guide.html, media-kit.html, press-mentions.html, senior-engineer-ai-fatigue.html, vibe-coding-ai-fatigue.html

**Root cause:** These pages linked to `href="style.css"` which doesn't exist in the root directory. The correct file is `css/style.min.css` (66KB minified). These pages were rendering without any site-wide CSS — broken nav, footer, dark mode, cards, etc.

**Fix:** All 8 pages now use `<link rel="preload" href="css/style.min.css" as="style" onload="this.rel='stylesheet'" />` pattern.
