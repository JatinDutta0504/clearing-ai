### Task 2: Font Loading Fix (145 pages)

**Critical issue found:** 145 out of 167 HTML files had broken Google Fonts `onload` handlers.

**Root cause:** The non-render-blocking font loading pattern used `onload="this.media=\'all\'"` (backslash-escaped single quotes inside a double-quoted HTML attribute). This is invalid HTML — browsers treat `\'` as a literal backslash+quote sequence, not as an escaped quote, causing the `onload` JavaScript handler to never fire, making Google Fonts render-blocking.

**Fix applied:** All 145 affected files fixed using byte-level replacement:
- BROKEN: `onload="this.media=\'all\'"` (backslash before each quote)
- FIXED: `onload="this.media='all'"` (plain single quotes)

**Impact:** Fonts now load asynchronously (non-render-blocking). Expected LCP improvement on pages with Google Fonts.

**Files fixed:** 145 HTML files across the entire site.

2. **heading-order** — H4 elements appear after H1 without H2/H3 parents. Specifically in tracker sections (ai-detox-plan, daily-practice, checkin, journal). 3. **list / listitem** — Nav dropdown markup: `<ul>` inside `<li>` without explicit list item markers in some contexts.

4. **target-size** — Nav dropdown touch targets: links inside dropdown menus are <48px touch area.

5. **link-name** — Icon-only social links without accessible text labels (aria-label or visually hidden text).

**Recommended fixes (Priority order):**
