## What Was Found & Fixed

### Bug 1: 404 on `style.css?v=1.0` (quiz-results-tier-1 + tier-2)
- **Symptom:** Console error `Failed to load resource: the server responded with a status of 404 ()`
- **Root cause:** Both tier result pages had inline CSS in `<style>` block but also included a link to `style.css?v=1.0` — file doesn't exist
- **Pages affected:** `quiz-results-tier-1.html`, `quiz-results-tier-2.html`
- **Fix:** Removed `<link rel="stylesheet" href="style.css?v=1.0">` from both pages
- **Impact:** Console errors eliminated, browser no longer wastes a request on 404
- **Commits:** `8f30c18` (tier-1 fix), `03b86be` (tier-2 font loading fix)

### Bug 2: Blocking Google Fonts (quiz-results-tier-1 + tier-2)
- **Symptom:** LCP 3.9s FAIL vs index at 1.6s (3.2s gap from same setup)
- **Root cause:** Tier pages load Google Fonts (Literata + DM Sans) as blocking stylesheet
- **Fix:** Applied `media="print" onload="this.media='all'"` + `<noscript>` fallback (same pattern as index)
- **Expected impact:** LCP improvement from 3.9s → ~1.5s (matches index performance)
- **Pages verified clean:** All 57 other pages already use non-render-blocking font pattern ✅
- **Commit:** `03b86be`
