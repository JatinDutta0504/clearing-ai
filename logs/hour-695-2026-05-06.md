## This Window: CLS Fix + New Pillar Page

### Critical Technical Fix: CLS=1.0 Root Cause Eliminated

**The Problem:** All 186 pages used `<link rel="stylesheet" href="css/style.min.css" media="print" onload="this.media='all'" />` — the classic non-render-blocking CSS trick. When the browser switches the stylesheet from `media="print"` to `media="all"` on load, it triggers a full-page re-layout (CLS).

**The Fix:** Replaced ALL 101 files with `media="print" onload` pattern with clean `<link rel="stylesheet" href="css/style.min.css" />`.

**Also fixed:**
- Removed duplicate stylesheet links on 41 files (preload tag + main link causing doubles)
- Removed preload tags for style.min.css on 4 pages (preload=style caused LCP delay)
- Standardized 109 files to single correct stylesheet link
- Fixed 7 files still referencing `/css/style.css` (old non-minified path)

**Files updated:** 121 files changed in git commit
