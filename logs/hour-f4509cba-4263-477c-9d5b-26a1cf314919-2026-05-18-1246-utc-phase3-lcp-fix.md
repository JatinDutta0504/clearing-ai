## What Was Built

**LCP fix — Google Fonts render-blocking across all pages**

### The Problem
index.html LCP was **3536ms** (score: 81, needs <2500ms to pass Core Web Vitals). Font loading was synchronous and blocking first paint.

### Root Cause
Google Fonts `<link rel="stylesheet">` was render-blocking — browser could not paint until fonts loaded. recovery.html and ai-detox-plan.html didn't have this issue (they had LCP ~2000ms already).

### The Fix
Python script (`/tmp/fix-font-blocking.py`) updated **161 pages** with async font loading pattern:
```html
<!-- BEFORE (render-blocking) -->
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter..." />

<!-- AFTER (non-render-blocking) -->
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter..."
      media="print" onload="this.media='all'" />
<noscript><link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter..." /></noscript>
```

Inline `@font-face` fallbacks in `<head>` ensure readable text during font load (no blank screen/flash).
