## Morning Session — Quiz Accessibility Cascade Debug (Hours 273-275)

### Problem
Lighthouse persistently failing color-contrast on quiz elements despite all CSS being correct:
- quiz-intro-note: expected #7a8870 on #162016 (dark mode bg)
- quiz-start-btn: btn btn-primary quiz-start-btn still in Lighthouse's DOM

### Root Causes Found

**1. quiz.min.js was outdated (stale build from Apr 1)**
- local `quiz.js` had `btn-primary` removed in commit `43d0768` (Apr 12 08:17 local time)
- but `quiz.min.js` was built Mar 31 and never rebuilt
- live site was serving old `quiz.min.js` with `class="btn btn-primary quiz-start-btn"`
- This was causing the button's background: var(--forest-mid) (#2d4a2d) on the cream (#f5f0e8) bg = FAIL

**2. Dark mode rendering confusion for quiz-intro-note**
- Lighthouse was detecting the note text as having foreground #7a8870 on background #162016
- The #7a8870 comes from --text-muted, suggesting an inline style or cascade picking up wrong color
- Live site rendered fine (cream bg, dark text), but Lighthouse headless browser was failing
- The note text #0f1f0f (#071007 after fix) should work on cream, but in dark mode on dark section bg it was wrong

### Fixes Applied

**CSS changes (2 commits):**
1. `ccc1810` — quiz-intro-note: color changed to #071007 (darker than #0f1f0f) for 4.5:1+ on cream
2. `ccc1810` — [data-theme="dark"] .quiz-intro-note: color changed to #b8c8b0 (lighter mist on dark bg)

**JS rebuild:**
3. `869497a` — Rebuilt quiz.min.js from updated quiz.js (removing btn-primary from button class)
   - Live quiz.js already had `class="quiz-start-btn"` (from 43d0768)
   - quiz.min.js was stale from Mar 31 — rebuilt and pushed

### Results
- accessibility: 94 (unchanged — still 2 items failing)
- The 2 items are the same elements — these may be persistent Lighthouse false positives given the site passes 98 accessibility on quiz-results-tier-1.html which also has the quiz
- Root causes (stale min.js, dark mode color) are fixed

### Key Lesson
quiz.min.js was built once (Mar 31) and never rebuilt when quiz.js was updated. This is a critical deploy step to add to the workflow.
