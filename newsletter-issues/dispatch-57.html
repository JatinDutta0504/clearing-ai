The Confirmation Loop

How AI use begets more AI use — and how to break it

The Clearing · Weekly for software engineers · ~6 min read

You've noticed it. Every task starts to feel like it needs AI. Not because you're dependent — because the muscle for doing it yourself has quietly atrophied. So you reach for the shortcut. The shortcut works. You reach faster next time. The loop closes.

The loop works like this

1.You use AI to solve a problem → it solves it faster than you expected
2.Your brain registers this as "I couldn't have done it that fast" → subtle competence doubt starts here
3.Next problem: you reach for AI before even trying → velocity reinforces the pattern
4.The small attempts stop. Those small attempts were where skill lived.
5.Now every task that doesn't use AI feels slow and uncertain
6.You're locked in. Not because AI is bad. Because the loop never let you practice.
60/40
If you use AI for 60% of your coding tasks, you're practicing 40% of your previous skill. That 40% decays. The 60% grows. But that 40% — the debugging, the architecture decisions, the edge cases — that's where senior engineers live. That's where the work feels like yours.

What breaking the loop actually looks like

The hardest part isn't the no-AI block. It's the first week of feeling slow. Actually slow. Like you're learning again.

That's the point.

The discomfort is the signal that you're rebuilding something. Not performing something. Rebuilding it from the ground up.

"I spent three years using AI for almost everything. When I finally tried to build something without it — a small API for a side project — I couldn't debug a simple 500 error for two days. I used to debug that in twenty minutes. I didn't realize how much I'd lost until I tried to use it."

— Senior IC, 9 years experience, anonymous submission

This week's challenge:
One intentional, small, AI-free task. Not a big project. Not a week off. One afternoon where you let yourself be slow on purpose. Build something simple. Debug something without asking AI first. The friction is the point.
